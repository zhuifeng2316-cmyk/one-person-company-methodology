# 完美主义是披着美德外衣的拖延症

你可能不承认自己是拖延症。
"我不是拖延，我只是想做到最好。"

这句话，是完美主义最常见的伪装。

它让你觉得自己在努力，其实是在逃避。

## 完美主义的本质

完美主义不是对质量的追求。
是对失败的恐惧。

真正追求质量的人，接受"够好就行"。
完美主义者不接受"够好"。永远不够好。所以永远不出发。

区别在哪？
一个在做。一个在准备。

准备多久算够？答案是——没有够。准备永远不够。

Vigma的"7天验证法"为什么有效？因为它强制你停止准备，开始行动。七天之内，不管做成什么样子，必须有一个可展示的东西。逼自己出丑，是对付完美主义的最好武器。

## 完美主义如何毁掉一人公司

一人公司最怕的是什么？

不是产品不够好。
不是内容不够好。
是根本没有产品，根本没有内容。

完美主义的一人公司创始人，永远在"打磨"。
打磨了半年，一个用户都没有。因为没发布过。

发布一个80分的产品，用户会告诉你哪里可以改进。
等待一个100分的产品，你永远等不到。

## 完美主义者的大脑运作方式

收到负面反馈 → 焦虑 → 继续打磨 → 继续不发布 → 继续没反馈 → 循环

打破这个循环的方法只有一个：**强制发布**。

哪怕只写200字，先发出去。
哪怕只有一个功能，先上线。
哪怕只做了一版，先给10个人看。

10个人里，只要有1个说"这个对我有用"，你就赢了。

## 3步对付完美主义

**第1步：给每个任务设deadline。**
没有截止日期的"做到最好"等于"永远做不完"。

**第2步：发布前最多改两遍。**
第一遍写，第二遍改，第三遍删减。超过两遍就发。

**第3步：把"发布"定义为成功。**
不要把"完美"定义为成功。在一人公司，"发布"本身就是胜利。

## 一个问题

如果明天必须发布，你还需要改什么？

列出来。
只有那些列出来的，才是真正重要的。

剩下的，现在就发。
