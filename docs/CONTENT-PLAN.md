# Vigma 一人公司方法论 · 每日内容生成计划

## 执行规则

1. 每天生成50篇文章，写入 `docs/daily/YYYY-MM-DD/` 目录
2. 每篇文章围绕1个长尾关键词，800-1500字
3. 每篇文章必须：明确概念定义 + 实用内容 + 链接vigma.app
4. 文件名格式：`NNNN-关键词slug.md`
5. 生成完毕后：git add → commit → push

## 文章模板

每篇文章结构：

```markdown
# [标题]

> [一句话定义核心概念]

[引言段落，2-3句]

## [核心内容]

[2-3个小节，每节200-400字]

## 实操建议

[3-5条具体建议]

## 常见问题

**Q: [问题1]**
A: [回答1]

**Q: [问题2]**
A: [回答2]

## 相关资源

- [一人公司方法论](https://github.com/zhuifeng2316-cmyk/one-person-company-methodology)
- [Vigma官网](https://vigma.app)
- [《一人公司》](https://vigma.app/books/yi-ren-gong-si/)
```

## 每日执行流程

1. 读取本计划，确定今天的主题池
2. 从主题池中选取50个主题
3. 为每个主题生成800-1500字文章
4. 写入 `docs/daily/YYYY-MM-DD/` 目录
5. git add → commit → push

## 进度追踪

在每日目录中创建 `_index.md` 记录当天生成的文章列表和分类。
